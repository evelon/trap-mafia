# 피처 스펙: 방 화면 UI (Room Screen)

## 사용자 시나리오 및 테스트

### 사용자 스토리 1 - 방 상태 실시간 확인

게임 방에 입장한 유저는 현재 방의 상태를 실시간으로 확인할 수 있다. 방 이름, 방장이 누구인지, 현재 참가자 목록, 방 설정(최대 인원, 각종 시간 제한 등)이 화면에 표시된다. 다른 유저가 방에 입장하거나 퇴장하면 즉시 참가자 목록이 갱신된다.

**인수 시나리오**:

1. **Given** 유저가 게임 방 화면에 접속했을 때, **When** 화면이 로드되면, **Then** 방 이름, 방장 정보, 참가자 목록, 방 설정이 표시된다
2. **Given** 유저가 방 화면을 보고 있을 때, **When** 다른 유저가 방에 입장하면, **Then** 해당 유저가 참가자 목록에 즉시 추가된다
3. **Given** 유저가 방 화면을 보고 있을 때, **When** 다른 유저가 방을 나가면, **Then** 해당 유저가 참가자 목록에서 즉시 제거된다

---

### 사용자 스토리 2 - 게임 시작

MVP에서는 모든 참가자가 호스트 권한을 가진다. 따라서 누구나 게임을 시작할 수 있다. 이미 게임이 진행 중일 때는 게임 시작 버튼이 표시되지 않는다.

**인수 시나리오**:

1. **Given** 유저가 방 화면에 있고 게임이 진행 중이 아닐 때, **When** "게임 시작" 버튼을 누르면, **Then** 게임 시작 요청이 전송되고 SSE로 `room.case.started` 이벤트를 수신하면 게임 화면으로 이동한다 (모든 참가자가 동일한 시점에 이동)
2. **Given** 게임이 이미 진행 중일 때, **When** 유저가 방에 접속하면, **Then** 게임 화면으로 이동한다 (게임 화면 UI는 04 스펙에서 정의)

---

### 사용자 스토리 3 - 방 나가기

방에 있는 유저는 언제든 방을 나갈 수 있다. 나가기를 실행하면 로비 화면으로 돌아간다.

**인수 시나리오**:

1. **Given** 유저가 방 화면에 있을 때, **When** "방 나가기" 버튼을 누르면, **Then** 퇴장 요청이 전송되고 성공 시 로비 화면으로 이동한다

---

### 엣지 케이스

- 실시간 연결이 끊기면 어떻게 되는가? → 연결 오류 상태를 화면에 표시하고 재연결을 시도한다. 일정 횟수/시간 후에도 재연결에 실패하면 유저에게 새로고침 또는 로비 이동 버튼을 표시한다
- 게임 시작에 실패하면 어떻게 되는가? → 에러 메시지를 표시하고 방 화면에 머문다

## 요구사항

### 기능 요구사항

- **FR-001**: 방 화면에는 방 이름이 표시되어야 한다
- **FR-002**: 참가자 목록이 표시되어야 한다
- **FR-003**: 방 설정이 표시되어야 한다 (UX/UI 설계상 일부 항목은 상세 보기 등으로 숨길 수 있다)
- **FR-004**: 참가자의 입장·퇴장이 실시간으로 목록에 반영되어야 한다
- **FR-005**: 호스트에게 "게임 시작" 버튼이 표시되어야 한다 (MVP에서는 모든 참가자가 호스트)
- **FR-006**: 게임이 이미 진행 중이면 게임 화면으로 이동한다 (세부 UI는 04 스펙에서 정의)
- **FR-007**: 모든 참가자에게 "방 나가기" 버튼이 표시되어야 한다
- **FR-008**: 실시간 연결 오류 발생 시 사용자에게 오류 상태를 알려야 한다

## 가정사항

- 방 화면은 `/rooms/current` 경로에 해당하며, 방에 참가 중이지 않은 유저는 접근할 수 없다 (02 스펙의 라우팅 가드가 처리)
- MVP에서는 모든 참가자가 호스트 권한을 가진다. 게임 시작 등 호스트 액션을 누구나 실행할 수 있다
- 레디/언레디 기능은 MVP 범위 외로 구현하지 않는다
- 유저 강제 퇴장(kick) UI는 이 스펙에서 다루지 않는다 (별도 스펙 예정)
- 방 설정은 읽기 전용으로 표시한다 (설정 변경 기능은 MVP 범위 외)
