# Specify — 피처 스펙 문서 생성 스킬

`/specify` 뒤에 전달된 입력이 피처 설명이다. 텍스트뿐 아니라 이미지(와이어프레임, 스크린샷, 다이어그램 등)가 함께 첨부될 수 있다. 모든 입력을 종합하여 구조화된 스펙 문서를 생성한다.

## 실행 흐름

### 1. 피처 설명 파싱

`/specify` 뒤의 입력에서 핵심 개념을 추출한다.
- 텍스트: 행위자(actors), 행동(actions), 데이터(data), 제약조건(constraints) 식별
- 이미지: 화면 구성, UI 요소, 흐름, 레이아웃 등을 분석하여 요구사항 도출
- 피처 설명이 비어있으면 에러: "피처 설명이 제공되지 않았습니다"

### 2. Short Name 생성 및 확인

피처 설명에서 2~4 단어의 짧은 이름 후보를 생성하여 사용자에게 제시한다.

규칙:
- 영문 kebab-case 사용
- 액션-명사 형태 권장
- 기술 용어/약어 보존 (OAuth2, API, JWT 등)

### 3. 디렉토리 생성 (자동 넘버링)

`specs/` 하위 디렉토리를 스캔하여 다음 번호를 자동 할당한다.

```
specs/
├── 01_auth/
├── 02_room/
└── 03_realtime-chat/   ← 새로 생성 (다음 번호 자동 할당)
    ├── spec.md
    └── checklists/
        └── requirements.md
```

넘버링 규칙:
- 기존 디렉토리에서 `{숫자}_` 패턴의 가장 큰 번호를 찾아 +1
- 두 자리 zero-padding (01, 02, ... 99)
- `backup_` 등 번호가 없는 디렉토리는 무시

### 4. 스펙 문서 작성

아래 템플릿 구조에 따라 `spec.md`를 **한국어**로 작성한다.

#### 작성 원칙

- **WHAT과 WHY에 집중**: 사용자가 무엇을 필요로 하고 왜 필요한지
- **HOW는 배제**: 기술 스택, API, 코드 구조, 프레임워크 언급 금지
- **비즈니스 관점**: 비개발 이해관계자도 이해할 수 있는 수준
- **스펙 안에 체크리스트를 넣지 않음**: 체크리스트는 별도 파일

#### 합리적 기본값 (이런 것들은 질문하지 않고 기본값 사용)

- 데이터 보존: 해당 도메인의 업계 표준
- 성능 목표: 일반적인 웹/모바일 앱 수준
- 에러 처리: 사용자 친화적 메시지 + 적절한 폴백
- 인증 방식: 웹 앱 표준 세션 기반 또는 OAuth2
- 연동 패턴: 프로젝트에 적합한 패턴

#### 불명확한 부분 처리

- 맥락과 업계 표준을 기반으로 합리적 추론 우선
- `[확인 필요: 구체적 질문]` 마커는 **최대 3개**까지만 사용
- 마커 사용 기준: 범위에 큰 영향 / 합리적 해석이 여러 개 / 기본값이 없는 경우
- 우선순위: 범위 > 보안/개인정보 > 사용자 경험 > 기술 세부사항

## 5. 스펙 품질 검증

스펙 작성 후 `{feature_dir}/checklists/requirements.md`에 품질 체크리스트를 생성하고 검증한다.

`templates/checklist.md` 파일을 읽어서 구조를 따른다.

### 검증 프로세스

1. 체크리스트의 각 항목을 스펙과 대조하여 통과/실패 판정
2. 실패 항목이 있으면 스펙을 수정하고 재검증 (최대 3회)
3. `[확인 필요]` 마커가 남아있으면 사용자에게 옵션 테이블로 제시:

```markdown
## 질문 [N]: [주제]

**맥락**: [관련 스펙 섹션 인용]

**확인이 필요한 사항**: [구체적 질문]

**제안 답변**:

| 옵션 | 답변 | 영향 |
|------|------|------|
| A    | [첫 번째 제안] | [피처에 미치는 영향] |
| B    | [두 번째 제안] | [피처에 미치는 영향] |
| C    | [세 번째 제안] | [피처에 미치는 영향] |
| 직접 입력 | 직접 답변 제공 | [설명] |
```

- 질문은 Q1, Q2, Q3 순서로 번호 매기고 최대 3개
- 모든 질문을 한 번에 제시
- 사용자 응답 후 스펙 업데이트 및 재검증

## 6. 완료 보고

모든 작업이 끝나면 다음을 보고한다:
- 생성된 디렉토리 경로
- 스펙 파일 경로
- 체크리스트 결과 요약
- `[확인 필요]` 마커가 있다면 해당 질문 제시
